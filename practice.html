Создание сайта
в VS Code

Пошаговое руководство по разработке информационного сайта — от установки редактора до публикации готового проекта.

4.1 Подготовка рабочего окружения

Прежде чем приступить к созданию сайта, необходимо установить и настроить инструменты разработки.

1

Скачайте и установите VS Code

Перейдите на официальный сайт code.visualstudio.com, выберите версию для вашей операционной системы (Windows / macOS / Linux) и установите редактор.

2

Установите расширение Live Server

В VS Code откройте панель расширений (Ctrl+Shift+X), найдите «Live Server» и нажмите «Установить». Это расширение запускает локальный сервер с авто-обновлением при сохранении файлов.

3

Создайте папку проекта

Создайте новую папку на диске (например, my-website). Откройте её в VS Code через меню Файл → Открыть папку.

4

Создайте структуру файлов

В папке проекта создайте файлы: index.html, style.css, main.js и подпапку images/.

Структура проекта:

СТРУКТУРА ФАЙЛОВ
my-website/
├── index.html          # Главная страница
├── about.html          # Страница «О нас»
├── articles.html       # Страница статей
├── contact.html        # Контакты
├── style.css           # Общие стили
├── main.js             # JavaScript
└── images/             # Папка изображений
    ├── logo.png
    └── hero-bg.jpg

4.2 Создание HTML-структуры

Начнём с главного файла index.html. В VS Code создайте файл и введите !, затем нажмите Tab — редактор автоматически вставит базовый шаблон HTML5.

index.html — Базовая структура
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>Мой информационный сайт</title>
  <link rel="stylesheet" href="style.css"/>
</head>
<body>

  <!-- Шапка сайта -->
  <header class="site-header">
    <div class="container">
      <a href="index.html" class="logo">МойСайт</a>
      <nav>
        <ul>
          <li><a href="index.html">Главная</a></li>
          <li><a href="articles.html">Статьи</a></li>
          <li><a href="about.html">О нас</a></li>
          <li><a href="contact.html">Контакты</a></li>
        </ul>
      </nav>
    </div>
  </header>

  <!-- Главный контент -->
  <main>
    <section class="hero">
      <h1>Добро пожаловать на сайт</h1>
      <p>Здесь вы найдёте полезную информацию.</p>
      <a href="articles.html" class="btn">Читать статьи</a>
    </section>
  </main>

  <!-- Подвал -->
  <footer>
    <p>© 2026 МойСайт. Все права защищены.</p>
  </footer>

  <script src="main.js"></script>
</body>
</html>

4.3 Написание CSS-стилей

В файле style.css задаём внешний вид сайта. Начинаем с обнуления стандартных стилей браузера и определения глобальных переменных.

style.css — Основные стили
/* Обнуление и базовые стили */
*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

/* CSS-переменные для единого стиля */
:root {
  --color-primary:    #2563eb;
  --color-bg:         #ffffff;
  --color-text:       #1e293b;
  --color-muted:      #64748b;
  --font-main:        'Segoe UI', sans-serif;
  --container-width:  1100px;
  --radius:           8px;
}

body {
  font-family:  var(--font-main);
  color:        var(--color-text);
  line-height:  1.6;
  background:   var(--color-bg);
}

/* Контейнер для центрирования */
.container {
  max-width: var(--container-width);
  margin:    0 auto;
  padding:   0 1.5rem;
}

/* Шапка сайта */
.site-header {
  background:     var(--color-primary);
  padding:        1rem 0;
  position:       sticky;
  top:            0;
  z-index:        100;
}

/* Адаптивная сетка карточек */
.cards-grid {
  display:                grid;
  grid-template-columns:  repeat(auto-fit, minmax(280px, 1fr));
  gap:                    1.5rem;
}

/* Адаптивность */
@media (max-width: 600px) {
  .site-header nav {
    display: none; /* скрываем на мобильных */
  }
}

4.4 Добавление интерактивности (JavaScript)

В файле main.js реализуем интерактивные элементы: мобильное меню, плавную прокрутку и форму обратной связи.

main.js — Интерактивность
// Мобильное меню (бургер-кнопка)
const burger = document.getElementById('burger');
const navMenu = document.getElementById('navMenu');

burger.addEventListener('click', () => {
  navMenu.classList.toggle('open');
});

// Закрываем меню при клике на ссылку
document.querySelectorAll('.nav-link').forEach(link => {
  link.addEventListener('click', () => {
    navMenu.classList.remove('open');
  });
});

// Плавное появление элементов при скролле
const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
    }
  });
}, { threshold: 0.1 });

document.querySelectorAll('.fade-in').forEach(el => {
  observer.observe(el);
});

// Форма обратной связи
const contactForm = document.getElementById('contactForm');

if (contactForm) {
  contactForm.addEventListener('submit', function(e) {
    e.preventDefault();

    const name = document.getElementById('name').value.trim();
    const email = document.getElementById('email').value.trim();

    if (!name || !email) {
      alert('Пожалуйста, заполните все поля');
      return;
    }

    console.log(`Отправлено: ${name}, ${email}`);
    alert('Сообщение отправлено! Спасибо.');
    contactForm.reset();
  });
}

4.5 Тестирование сайта

После написания кода необходимо провести комплексное тестирование.

1

Запустите Live Server

Правой кнопкой мыши кликните на файл index.html в VS Code и выберите «Open with Live Server». Сайт откроется по адресу http://127.0.0.1:5500.

2

Проверьте кроссбраузерность

Откройте сайт в Chrome, Firefox, Edge и Safari. Убедитесь, что все элементы отображаются корректно.

3

Проверьте адаптивность

В Chrome DevTools (F12) откройте режим «Toggle device toolbar» (Ctrl+Shift+M) и проверьте отображение на смартфоне и планшете.

4

Валидация кода

Проверьте HTML на сайте validator.w3.org, CSS — на jigsaw.w3.org/css-validator. Исправьте все найденные ошибки.

5

Публикация

Загрузите файлы на хостинг (например, через GitHub Pages или Netlify). Настройте домен и SSL-сертификат для безопасного соединения HTTPS.

💡 Рекомендация: При публикации на GitHub Pages достаточно создать репозиторий, загрузить файлы и включить GitHub Pages в настройках. Сайт станет доступен по адресу username.github.io/repository-name — бесплатно и без настройки сервера.

4.6 Рекомендации по дальнейшему развитию

  • Подключите Google Analytics для отслеживания посещаемости
  • Оптимизируйте изображения (формат WebP, сжатие)
  • Настройте мета-теги для SEO: <meta name="description">, Open Graph
  • Добавьте файл sitemap.xml для поисковых систем
  • Настройте файл robots.txt
  • Регулярно обновляйте контент для поддержания актуальности
